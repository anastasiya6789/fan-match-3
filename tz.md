### Техническое задание (ТЗ) для веб-игры "Три в ряд" (Match-3)

**Описание проекта:**  
Веб-приложение — простая игра в стиле match-3 (три в ряд), где игрок соединяет одинаковые элементы на сетке для набора очков и прохождения уровней. Игра будет развернута на GitHub Pages для бесплатного доступа по ссылке. Поддержка до 5 игроков: логин, сохранение прогресса (уровни, очки, монетки), рейтинг друзей (топ по очкам). Элементы (кубики) — цветные с кастомными изображениями (твои PNG-файлы). Добавлены бустеры (покупаемые за монетки, но монетки бесконечные и не ограничивают игру). Ограничение на шаги per уровень, но при исчерпании — бесплатный выбор: +10 шагов или перезапуск. После победы на уровне играет конкретная песня (укажи её MP3-файл или URL). Без фокуса на безопасности данных (открытый доступ в БД). Стек: VS Code, HTML/JS/CSS, Phaser.js для игры, Supabase для аутентификации и хранения данных (бесплатно), GitHub для репозитория и хостинга.

**Требования к стеку и инструментам (в VS Code):**  
- Языки: HTML5, CSS3, JavaScript (vanilla).  
- Библиотеки: Phaser.js (для игры), Supabase JS SDK (для логина/данных).  
- Extensions в VS Code: Live Server, ESLint, Prettier, GitHub Pull Requests.  
- Хостинг: GitHub Pages (frontend), Supabase (backend, free tier).  
- Ассеты: Кастомные PNG для кубиков (4-6 типов элементов), MP3 для песни победы.  
- Всё бесплатно, без установки npm (используй CDN для библиотек).

#### Список реализуемых функций
1. **Регистрация и логин:**  
   - Форма для ввода email/пароля (или GitHub/Google логин через Supabase).  
   - После логина: Сохранение имени (логин) в профиле.  
   - Автоматический вход при повторном посещении (если сессия активна).

2. **Игра match-3:**  
   - Сетка 8x8 (или настраиваемая) с кастомными кубиками: 4-6 типов, каждый — цветной квадрат с твоим изображением (PNG, загружены в проект).  
   - Механика: Свайп/клик для свапа соседних элементов, матчинг 3+ в ряд/столбик → удаление, падение новых, комбо для бонусов.  
   - Очки: Начисляются за матчи (базовые + множители за комбо).  
   - Уровни: 5-10 уровней с растущей сложностью (цели по очкам, больше элементов).  
   - Ограничение шагов: Фиксное на уровень (например, 20-50 шагов). При 0 шагов: Модалка с выбором — "Переиграть уровень" или "Получить +10 шагов бесплатно" (бесконечно повторяемо, без ограничений).  
   - Прогресс: Сохранение текущего уровня, очков, монеток в Supabase после каждого уровня.

3. **Бустеры:**  
   - 3-4 типа: Например, "Бомба" (удаляет область), "Перемешать" (shuffle сетки), "Доп. шаг" (+5 шагов).  
   - Покупка: За монетки (кнопка "Купить" в магазине перед уровнем).  
   - Монетки: Стартовые 10000+, начисляются за уровни (много, например +1000 per уровень). Не ограничивают — если кончатся, можно заработать или игнорировать (играй честно без бустеров).  
   - Вайб: Опционально — используй для фана, не обязательно для прохождения.

4. **Песня после победы:**  
   - После прохождения уровня: Проигрывание конкретной песни (MP3-файл, загружен в проект или URL). Авто-стоп при старте следующего уровня.  
   - Используй HTML5 Audio API в JS.

5. **Рейтинг друзей:**  
   - Таблица топ-5 по очкам (имя + очки, обновляется realtime).  
   - Только для залогиненных (друзья — все пользователи в БД, но max 5).  
   - Отображение: На главной странице или после уровня.

6. **Сохранение данных:**  
   - В Supabase: Профиль (имя, уровень, очки, монетки, бустеры).  
   - Realtime синхронизация для рейтинга.

7. **Интерфейс и UI:**  
   - Главная: Логин, старт игры, рейтинг.  
   - Игровой экран: Сетка, счётчик шагов/очков/монеток, кнопки бустеров.  
   - Мобильный-friendly (responsive CSS).  
   - Нет рекламы/платежей — всё бесплатно.

8. **Деплой:**  
   - Залить на GitHub, доступ по ссылке (GitHub Pages).

**Ограничения:**  Простая логика, без безопасности (открытые правила в Supabase).
